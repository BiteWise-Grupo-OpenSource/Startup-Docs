# Capítulo III: Requirements Specification 

## 3.1 User Stories

| Epic / Story ID | Título | Descripción | Criterios de Aceptación | Relacionado con (Epic ID) |
| --- | --- | --- | --- | --- |
| EP001 / US001 | Registro de Usuario Nuevo | Como usuario nuevo, quiero registrarme con mi correo electrónico para crear una cuenta y acceder a la plataforma. | Escenario 1: Email válido<br>Dado que se proporciona un email válido,<br>Cuando se solicita el registro,<br>Entonces se crea una cuenta y se concede acceso.<br><br>Escenario 2: Email inválido<br>Dado que se proporciona un email inválido,<br>Cuando se solicita el registro,<br>Entonces se muestra un mensaje de error. | EP001 |
| EP001 / US002 | Configuración de Perfil Básico | Como usuario registrado, quiero configurar mi perfil con datos básicos como edad, género y altura para que el sistema calcule mis necesidades nutricionales. | Escenario 1: Datos completos<br>Dado que se ingresa datos de perfil de usuario,<br>Cuando se guarda el perfil,<br>Entonces se calculan las necesidades nutricionales basadas en edad, género y altura. | EP001 |
| EP001 / US003 | Agregar Restricciones Dietéticas | Como usuario, quiero agregar restricciones dietéticas para que los planes de alimentación eviten alimentos incompatibles. | Escenario 1: Agregar restricción<br>Dado que se especifican restricciones dietéticas,<br>Cuando se generan planes de comidas,<br>Entonces se excluyen alimentos incompatibles.<br><br>Escenario 2: Sin restricciones<br>Dado que no se especifican restricciones,<br>Cuando se generan planes,<br>Entonces se incluyen todos los alimentos. | EP001 |
| EP001 / US004 | Establecer Metas de Salud | Como usuario, quiero establecer metas de salud para que los planes se adapten a mis objetivos. | Escenario 1: Meta definida<br>Dado que se establecen metas de salud,<br>Cuando se generan planes de comidas,<br>Entonces los planes se ajustan para cumplir las metas. | EP001 |
| EP002 / US005 | Generar Plan Semanal | Como usuario, quiero generar un plan de alimentación semanal basado en mis preferencias y metas para tener una guía clara de comidas. | Escenario 1: Preferencias definidas<br>Dado que se definen preferencias y metas,<br>Cuando se solicita la generación de plan,<br>Entonces se crea un plan semanal de comidas. | EP002 |
| EP002 / US006 | Personalizar Comidas Diarias | Como usuario, quiero personalizar comidas específicas de cada día dentro del plan semanal para adaptarlo a mis gustos diarios. | Escenario 1: Modificación diaria<br>Dado que existe un plan semanal,<br>Cuando se modifican comidas diarias,<br>Entonces el plan se actualiza con los cambios. | EP002 |
| EP002 / US007 | Ver Desglose Nutricional | Como usuario, quiero ver el desglose nutricional de cada comida para tomar decisiones informadas. | Escenario 1: Comida seleccionada<br>Dado que se selecciona una comida,<br>Cuando se solicitan detalles nutricionales,<br>Entonces se muestran calorías, proteínas y otros nutrientes. | EP002 |
| EP002 / US008 | Sugerencias de Alternativas | Como usuario, quiero recibir sugerencias de alternativas saludables cuando un ingrediente no esté disponible para mantener el equilibrio nutricional. | Escenario 1: Ingrediente faltante<br>Dado que un ingrediente no está disponible,<br>Cuando se solicitan alternativas,<br>Entonces se sugieren sustitutos saludables. | EP002 |
| EP002 / US009 | Ajustar por Actividad Física | Como usuario, quiero ajustar el plan según mi nivel de actividad física para optimizar mis necesidades calóricas. | Escenario 1: Nivel especificado<br>Dado que se especifica el nivel de actividad,<br>Cuando se ajusta el plan,<br>Entonces se recalcula la ingesta calórica. | EP002 |
| EP002 / US010 | Guardar Planes Favoritos | Como usuario, quiero guardar planes favoritos para reutilizarlos en el futuro sin generar uno nuevo cada vez. | Escenario 1: Plan creado<br>Dado que se crea un plan,<br>Cuando se solicita guardar,<br>Entonces el plan se almacena para uso futuro. | EP002 |
| EP003 / US011 | Registrar Consumo Diario | Como usuario, quiero registrar lo que como diariamente para que la app compare con el plan y me dé retroalimentación. | Escenario 1: Consumo registrado<br>Dado que se registra el consumo diario,<br>Cuando se solicita comparación,<br>Entonces se proporciona retroalimentación sobre adherencia. | EP003 |
| EP003 / US012 | Ver Gráficos de Progreso | Como usuario, quiero ver gráficos de mi progreso nutricional para motivarme a seguir mis metas. | Escenario 1: Datos disponibles<br>Dado que existen datos de progreso,<br>Cuando se solicitan gráficos,<br>Entonces se visualiza el progreso nutricional. | EP003 |
| EP003 / US013 | Calificar Comidas | Como usuario, quiero calificar las comidas del plan para que la app aprenda mis preferencias y mejore futuras sugerencias. | Escenario 1: Calificación enviada<br>Dado que se califica una comida,<br>Cuando se envía la calificación,<br>Entonces se registran las preferencias para sugerencias futuras. | EP003 |
| EP003 / US014 | Compartir Progreso | Como usuario, quiero compartir mi progreso con amigos o un entrenador para obtener apoyo social. | Escenario 1: Progreso compartido<br>Dado que se comparte el progreso,<br>Cuando se inicia el compartir,<br>Entonces los datos de progreso se envían a los destinatarios seleccionados. | EP003 |
| EP004 / US015 | Acceder a Artículos | Como usuario, quiero acceder a artículos sobre nutrición y salud para aprender más sobre alimentación saludable. | Escenario 1: Artículos disponibles<br>Dado que los artículos están disponibles,<br>Cuando se solicita acceso,<br>Entonces se muestran los artículos. | EP004 |
| EP004 / US016 | Ver Recetas Detalladas | Como usuario, quiero ver recetas detalladas con instrucciones paso a paso para preparar las comidas sugeridas. | Escenario 1: Receta seleccionada<br>Dado que se selecciona una receta,<br>Cuando se solicitan detalles,<br>Entonces se muestran ingredientes y pasos. | EP004 |
| EP004 / US017 | Generar Lista de Compras | Como usuario, quiero generar una lista de compras basada en el plan semanal para facilitar las compras en el supermercado. | Escenario 1: Plan semanal existe<br>Dado que existe un plan semanal,<br>Cuando se genera la lista de compras,<br>Entonces se listan los ingredientes. | EP004 |
| EP004 / US018 | Buscar Artículos por Tema | Como usuario, quiero buscar artículos por temas específicos para acceder a contenido relevante rápidamente. | Escenario 1: Tema de búsqueda<br>Dado que se ingresa un tema de búsqueda,<br>Cuando se realiza la búsqueda,<br>Entonces se devuelven artículos relevantes. | EP004 |
| EP005 / US019 | Seleccionar Idioma Español | Como usuario hispanohablante, quiero seleccionar el idioma español en la configuración para que toda la interfaz y planes estén en español. | Escenario 1: Español seleccionado<br>Dado que se selecciona español,<br>Cuando se establece el idioma,<br>Entonces la interfaz y planes están en español.<br><br>Escenario 2: Cambio de idioma<br>Dado que se cambia a español,<br>Cuando se recarga la página,<br>Entonces el contenido se traduce. | EP005 |
| EP005 / US020 | Seleccionar Idioma Inglés | Como usuario angloparlante, quiero seleccionar el idioma inglés en la configuración para que toda la interfaz y planes estén en inglés. | Escenario 1: Inglés seleccionado<br>Dado que se selecciona inglés,<br>Cuando se establece el idioma,<br>Entonces la interfaz y planes están en inglés. | EP005 |
| EP006 / US021 | Ver Beneficios en Landing Page | Como visitante, quiero ver los beneficios principales de la plataforma para entender su valor. | Escenario 1: Acceso a landing<br>Dado que se accede a la página de destino,<br>Cuando se ve la sección de beneficios,<br>Entonces se muestran las ventajas clave. | EP006 |
| EP006 / US022 | Explorar Características | Como visitante interesado en nutrición, quiero explorar las características disponibles para decidir si registrarme. | Escenario 1: Sección de características<br>Dado que se accede a la sección de características,<br>Cuando se navegan las características,<br>Entonces se muestran descripciones detalladas. | EP006 |
| EP006 / US023 | Contactar Soporte | Como visitante potencial, quiero contactar al soporte para resolver dudas antes de registrarme. | Escenario 1: Formulario de contacto<br>Dado que se solicita información de contacto,<br>Cuando se envía el formulario de contacto,<br>Entonces se envía la consulta. | EP006 |
| EP007 / TS001 | Endpoint para Generar Plan | Como desarrollador, quiero un endpoint RESTful para generar planes de alimentación. | Escenario 1: Solicitud exitosa<br>Dado que se envía una solicitud POST a /api/plans con preferencias,<br>Cuando se procesa la solicitud,<br>Entonces se devuelve un plan de comidas en formato JSON.<br><br>Escenario 2: Error en solicitud<br>Dado que se envía una solicitud POST inválida,<br>Cuando se procesa,<br>Entonces se devuelve un error 400. | EP007 |
| EP007 / TS002 | Endpoint para Obtener Perfil | Como desarrollador, quiero un endpoint para obtener datos de perfil de usuario. | Escenario 1: Autenticado<br>Dado que se envía una solicitud GET a /api/profile/{id},<br>Cuando se autentica,<br>Entonces se devuelven los datos de perfil de usuario. | EP007 |
| EP007 / TS003 | Endpoint para Actualizar Plan | Como desarrollador, quiero un endpoint para actualizar un plan existente. | Escenario 1: Actualización exitosa<br>Dado que se envía una solicitud PUT a /api/plans/{id} con cambios,<br>Cuando se autentica,<br>Entonces se actualiza el plan y se confirma. | EP007 |
